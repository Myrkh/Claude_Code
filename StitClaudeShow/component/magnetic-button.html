Magnetic Button

A button subtly attracted to the cursor when it's near. It's the kind of detail that most people won't notice consciously — but the page will feel more alive, and they won't know why.

hover near the button
spring 600ms · stiffness 1.6 · radius 160px

Drop-in. The button follows the cursor when it's within 160 px. Move your mouse out and it springs back. Pure pointer events, no RAF loop needed.

<script>
(function() {
  const stage = document.getElementById('stit-magnet-stage');
  const btn = document.getElementById('stit-magnet-btn');
  const inner = btn.querySelector('.inner');
  const RADIUS = 160;
  const STRENGTH = 0.35;

  stage.addEventListener('pointermove', (e) => {
    const rect = btn.getBoundingClientRect();
    const cx = rect.left + rect.width / 2;
    const cy = rect.top + rect.height / 2;
    const dx = e.clientX - cx;
    const dy = e.clientY - cy;
    const dist = Math.hypot(dx, dy);
    if (dist < RADIUS) {
      const pull = (1 - dist / RADIUS) * STRENGTH;
      btn.style.transform = `translate(${dx * pull}px, ${dy * pull}px)`;
      inner.style.transform = `translate(${dx * pull * 0.5}px, ${dy * pull * 0.5}px)`;
    }
  });
  stage.addEventListener('pointerleave', () => {
    btn.style.transform = '';
    inner.style.transform = '';
  });
})();
</script>

Apple-Craft

visionOS cursors, Dynamic Island, the dock — Apple is magnetic everywhere. Subtle pull is the opposite of heavy animation and it respects the user.

Editorial-Display

Awwwards-grade portfolios live on this effect. One magnetic CTA on a hero, restrained, and the page reads confident without shouting.

Playful-Warm

Arc Browser's tab interactions, Figma's canvas — magnetic pull makes the product feel personal. Go slightly stronger here (strength ~ 0.45) to lean into the playfulness.

  • ✕
    Pull strength of 1.0 (the button follows the cursor literally)

    The button glues itself to the pointer. Users can't click it because it's always where they aren't. The effect goes from subtle to slapstick.

    Instead: keep strength between 0.25 and 0.45. The button should suggest being pulled, not actually get pulled.

  • ✕
    No release spring on pointerleave

    When the cursor exits the stage, the button snaps back instantly. Looks like a glitch, not a feature.

    Instead: set transition: transform 600ms cubic-bezier(0.3, 1.6, 0.5, 1) on the button — the release springs back naturally.

  • ✕
    Magnetic everywhere

    Eight magnetic buttons on one page. The effect is now noise. Users see the motion coming and start ignoring it.

    Instead: one magnetic CTA per section. Use it for your primary call to action. Everything else is a regular button.

  • ✕
    Using a requestAnimationFrame loop instead of pointer events

    Running at 60 fps even when the cursor hasn't moved. Burns battery on laptops, wastes main thread.

    Instead: use pointermove. It fires only when the cursor actually moves. Zero idle cost.

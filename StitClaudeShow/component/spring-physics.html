Apple-Craft

Every iOS swipe, pull-to-refresh, and sheet transition uses springs. They're the signature of haptic-feeling UI without haptics.

Playful-Warm

Bouncy is the default here. A bouncy spring is what makes a stamp feel like a stamp instead of a fade. Restraint comes from picking only one bounce.

Industrial-Pro

Only smooth. A bouncing refresh icon in a Bloomberg terminal would be grounds for dismissal. Snap is acceptable on user-triggered feedback.

Power-Dense

Snappy only. Power users move fast; any overshoot feels like lag. Duration should be ≤ 150ms where possible.

  • ✕
    Using one ease everywhere

    ease-in-out for 12 different interactions. Entry gestures should feel different from exit gestures, and different again from feedback gestures.

    Instead: name three springs, use them consistently. Smooth for entry, snappy for user input, bouncy for reward only.

  • ✕
    Bouncing everything because bounces feel fun

    Every modal bounces in. Every toast bounces. Even the chevron bounces on hover. By the thirtieth bounce of the day, the user wants the app to stop moving.

    Instead: reserve bouncy for moments of celebration — success states, stamps, successful submit. Smooth for everything else.

  • ✕
    Tweening with ease-in-out and calling it a spring

    A spring has overshoot. If your curve's Y never exceeds 1, it's not a spring. cubic-bezier(0.2, 0, 0.3, 1) is a decelerate, not a spring.

    Instead: use real spring bezier values. cubic-bezier(0.45, 1.4, 0.55, 1) overshoots to 1.4 before settling. That's a spring.

  • ✕
    Ignoring prefers-reduced-motion

    A bouncing element is a nausea trigger for people with vestibular disorders. Ignoring the reduced-motion preference excludes those users from your product.

    Instead: wrap every spring in @media (prefers-reduced-motion: no-preference) { … }, or provide a reduced alternative that skips the overshoot.

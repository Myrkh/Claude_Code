Variable Font Playground

A single slider controlling font-variation-settings, wired live to a wordmark. The type doesn't snap between weights — it interpolates. That's the whole point of variable fonts, and almost nobody uses it.

<script>
(function() {
  const word = document.getElementById('stit-vf-word');
  const sW = document.getElementById('stit-vf-wght');
  const sS = document.getElementById('stit-vf-stop');
  const vW = document.getElementById('stit-vf-val-wght');
  const vS = document.getElementById('stit-vf-val-stop');
  function update() {
    const w = parseInt(sW.value, 10);
    const s = parseInt(sS.value, 10);
    word.style.setProperty('--wght', w);
    word.style.setProperty('--stop', s + '%');
    vW.textContent = w;
    vS.textContent = s + '%';
  }
  sW.addEventListener('input', update);
  sS.addEventListener('input', update);
  update();
})();
</script>

Editorial-Display

Type as image. Variable weight makes confident display type possible: a single font pulls light, regular, bold, black — no 4 webfont downloads.

Swiss-Quiet

Linear uses Inter variable at wght 450 for body, 550 for labels, 600 for emphasis. One font, three weights, zero tradeoff.

Apple-Craft

SF Pro is variable across wght and optical-size. Apple tunes opsz per glyph size for optical integrity. You should too.

  • ✕
    Loading Inter at 4 weights (400, 500, 600, 700) as static fonts

    Four @font-face declarations, four file downloads, no interpolation possible. Total bandwidth: ~160 KB. And you can't even animate between them.

    Instead: one variable font file covers the full range 100–900. ~55 KB total. font-variation-settings: "wght" 450 gets you a half-step if you want it.

  • ✕
    Using font-weight: 450 and hoping for the best

    font-weight is constrained to discrete stops in CSS. Half-steps may or may not interpolate depending on the browser and font.

    Instead: use font-variation-settings: "wght" 450 directly. It's the only way to reliably address specific interpolated values.

  • ✕
    Ignoring optical-size (opsz) when the font has it

    Same glyph at 12 px and 120 px needs different proportions to look right. Optical-size axis handles that. Almost nobody uses it.

    Instead: set font-optical-sizing: auto on html, or declare "opsz" explicitly for display headings.

  • ✕
    Animating font-weight and wondering why it jumps

    font-weight jumps in 100-unit steps. You get staircase animation.

    Instead: transition on font-variation-settings. It interpolates smoothly in any increment.

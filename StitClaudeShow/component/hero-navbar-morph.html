Note on restraint

The morph is invisible. That is the point.

Most site headers use one of two discrete patterns: a shrink-on-scroll (a step-function that snaps down to a smaller bar once a threshold is crossed), or a hide-on-scroll (the navbar disappears entirely when the user scrolls down, reappears on scroll up).

Both are bad. The first interrupts reading with a visual pop. The second loses the reader's anchor when they most need it — while searching for a menu item from deep inside a long document.

A good morph is not perceived as motion. It is perceived as a single object reshaping itself across the duration of a gesture that was happening anyway.
The four dimensions

Four continuous interpolations, one perceived motion.

The header drives four independent transforms from the same scroll progress (0 → 1): the header's own height (100dvh → 64px), the wordmark's scale and position (center → top-left), the secondary subtitle's opacity (1 → 0), and the nav items' emergence (0 → 1). Each uses the same --p CSS custom property.

On modern browsers, CSS scroll-driven animations (animation-timeline: view()) drive the property natively. On older engines, a single IntersectionObserver with requestAnimationFrame writes the same value. The fallback is visually identical.

Calibration

The morph duration is the hero height.

The scroll zone is sized at 100dvh. This means the entire morph completes exactly when the reader has scrolled past the initial viewport — the moment they've "left" the arrival page. Smaller zones feel abrupt. Larger zones feel dragged-out.

The calibration is not arbitrary: it's tied to the reader's expectation. The hero is earned by the scroll that carries them past it.

<!-- HTML -->
<header class="h2n-header">
  <div class="h2n-brand">
    <div class="line-1">Stit'Atelier</div>
    <div class="line-2">A studio for considered interfaces.</div>
  </div>
  <nav class="h2n-nav">
    <a href="#work">Work</a>
    <a href="#studio">Studio</a>
  </nav>
</header>
<div class="h2n-scroll-zone"></div>
<main>...content...</main>

/* CSS — driven by a single --p custom property, 0→1 */
.h2n-header {
  position: fixed; inset: 0 0 auto 0;
  height: 100dvh;
  --p: 0;
}
.h2n-brand {
  transform:
    translate(calc((1 - var(--p)) * -50% + 0px), calc(50vh * (1 - var(--p))))
    scale(calc(1 - var(--p) * 0.80));
  transform-origin: left center;
}
.h2n-nav { opacity: var(--p); }
.h2n-header.compact { height: 64px; }

/* MODERN · native scroll-driven animation */
@supports (animation-timeline: scroll()) {
  .h2n-scroll-zone { view-timeline-name: --morph; }
  .h2n-header {
    animation: morph linear forwards;
    animation-timeline: --morph;
    animation-range: entry 0% cover 100%;
  }
  @keyframes morph { to { --p: 1; } }
}

/* JS FALLBACK — one IntersectionObserver + rAF,
   writes the same --p variable on the header.
   See the demo source for the 20-line controller. */
— five ways to break it —

A morph that isn't invisible isn't a morph.

01 · DO NOT
Use a step function.
A navbar that snaps from large to small at a scroll threshold is the canonical failure. The reader's eye registers the step as a glitch. The morph must be continuous — every frame of scroll produces an intermediate state.
02 · DO NOT
Hide the navbar while scrolling down.
Medium-style hide-on-scroll patterns penalize the reader who knows what they want. The morph preserves the navbar at all times — the reader can always navigate without scrolling up.
03 · DO NOT
Use JavaScript to read scrollY on every frame.
CSS animation-timeline runs on the compositor thread at 60+ fps. Polling window.scrollY in onscroll runs on the main thread and will drop frames under load. Use CSS first, JS only as fallback.
04 · DO NOT
Shrink the wordmark below 12px.
A wordmark that becomes illegible when morphed is not a morph — it's a disappearance. The final scale must leave the wordmark readable at arm's length. 16px is the floor for most serifs.
05 · DO NOT
Skip the reversal.
Scrolling back up must reverse the morph at the same rate. If the navbar stays compact once the reader scrolls back to the top, the hero is betrayed. The morph must be symmetric in both directions.
Principles applied
Motion has a reasonEvery interpolation is driven by the reader's own gesture — scroll. Nothing animates on its own. As little design as possibleFour dimensions, one --p variable. Twelve lines of CSS carry the entire morph. HonestNo transform trickery hides the structure. The header is the navbar, at a different size. Long-lastingCSS scroll-driven animation shipped in Chrome 115, Safari 26. The component will outlive its JS fallback. Thorough to the last detailThe subtitle opacity uses a × 1.4 coefficient — it completes its fade before p reaches 1, so the compact state has no leftover fog.

<script>
  function stitSbToggle() {
    document.getElementById('stit-sb').classList.toggle('is-collapsed');

Power-Dense

Keyboard shortcuts visible on every item. Dense information density without feeling cramped. Linear's ur-sidebar lives here.

Swiss-Quiet

One accent (active indicator), hairline borders, zero decorative flourishes. The whole design vanishes so you can think about your work.

Apple-Craft

Spring-snappy width transition. Concentric radii (sidebar 14px → item 6px → icon 4px). Tooltips delayed 400ms. No effect has no reason.

Industrial-Pro

With icon colors drawn from project status (OKLCH-derived signal hues) and tabular nums on counters, the sidebar earns its place in a Bloomberg-grade tool.

  • ✕
    Emoji + icon redundancy (📁 Projects)

    An emoji next to a lucide icon next to the label. Three visual signals competing for the same word. Noise and ambiguity at once.

    Instead: one glyph per item. A stroked SVG icon for system items, a colored dot for user-defined things (projects, tags). Emojis belong to user content.

  • ✕
    Collapse that hides labels instantly

    Click collapse and all labels disappear in the same frame as the width shrinks. The sidebar looks like it crashed, not like it transformed.

    Instead: stagger label fade-outs with transition-delay per category (0ms shortcuts, 30ms labels, 60ms section heads). The cascade reads as choreography.

  • ✕
    Big logo at the top that takes 80 px

    The workspace logo occupies prime vertical space every day, telling the user information they already have. You know what product you opened.

    Instead: a compact workspace pill (36 px avatar + name + chevron) that doubles as the collapse trigger. It earns its pixels.

  • ✕
    No keyboard shortcut at all

    Power users live in the sidebar. Without ⌘1 / ⌘2 / ⌘3, they keep reaching for the mouse. Any app used daily must grow keyboard surface area.

    Instead: visible shortcuts on hover, trigger them from anywhere in the app, display them in a ⌘K palette too. Three entries for the same feature is a feature.

  • ✕
    Tooltips appearing immediately on hover in collapsed mode

    Move the cursor across the rail — 5 tooltips flash by in half a second. Visual panic instead of information.

    Instead: transition-delay: 400ms before the tooltip appears. The user has to hover intentionally. Reading the icon is the path of first resort, tooltip is the backup.

Principles shortcuts visible staggered label fade spring-snappy width active indicator as bar, not fill concentric radii 14 → 10 → 6 tooltip 400ms delay triple trigger (click · shortcut · button)

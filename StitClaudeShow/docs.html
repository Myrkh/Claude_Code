Stit'Claude / Docs

Install, use, extend.

Stit'Claude is a design discipline encoded as a Claude skill — a structured folder that gives any Claude model a direction when asked to design. This page shows how to install it, how to invoke it, and how to extend it with your own houses.

Install now View on GitHub
TL;DR

Clone the repo into your Claude skills directory. Claude will load it automatically. Invoke by asking "design with Stit'Claude" in any conversation.

#Install

One command. Three seconds. Stit'Claude lives as a folder in ~/.claude/skills/. When you ask Claude to design, it will pick up the skill automatically.

~/.claude/skills/ 
# Clone the skill into Claude's skills folder
git clone https://github.com/Myrkh/Master_Claude ~/.claude/skills/stitclaude

# Verify
ls ~/.claude/skills/stitclaude/SKILL.md
NOTE

Claude Desktop app users — the skills folder lives inside the app's data directory. On macOS: ~/Library/Application Support/Claude/skills/. On Windows: %APPDATA%\Claude\skills\. Claude Code users have the path above.

#Invoke the skill

Once installed, you don't load it manually. Claude reads the skill index on each conversation and activates Stit'Claude when it recognizes a design intent in your prompt.

For reliable activation, use one of these phrasings:

any conversation 
# Direct invocation
"Design a dashboard using Stit'Claude, house Industrial-Pro."

# Indirect — Claude picks the house itself
"Build me a landing page for a fashion brand — use Stit'Claude."

# Token-level invocation
"Generate an OKLCH palette with Stit'Claude principles."

#Anatomy

The skill folder is plain files — no build step, no runtime. Three top-level entries matter:

stitclaude/ ├── SKILL.md // the entry point Claude reads first ├── houses/ │   ├── editorial.md // one file per house │   ├── industrial-pro.md │   ├── swiss-quiet.md │   └── ... 5 more ├── references/ │   ├── tokens.css // OKLCH palette, motion, type scale │   ├── checklist.md // 12-point pre-ship checklist │   └── anti-patterns.md // 30 things the skill refuses └── examples/     ├── landing.html     ├── dashboard.html     └── ... more

Claude loads SKILL.md first. That file references the houses, the checklist, and the anti-patterns, and instructs Claude how to combine them on request.

#The 12-point checklist

Before shipping anything Stit'Claude-authored, run through these. Click each to mark as done — the state persists locally during your session.

  • Tokens defined in a single tokens.css, no literal colors in components.§ 1
  • Every radius is concentric: child = parent − padding.§ 2
  • Tabular numerics on every column of data.§ 3
  • All motion has a reason — remove anything you cannot justify.§ 4
  • Headlines balanced: text-wrap: balance on H1–H3.§ 5
  • Colors in OKLCH, never HSL, never hex for tokens.§ 6
  • APCA Lc ≥ 75 for body text. Checked, not estimated.§ 7
  • Dark mode tested, not derived. Every palette step verified.§ 8
  • Focus states visible on every interactive element.§ 9
  • Reduced motion respected: prefers-reduced-motion override.§ 10
  • One house only per artifact. No cross-contamination.§ 11
  • Something memorable: one detail the user will quote.§ 12

#Add your own house

Stit'Claude is designed to be forked. Each house is a single Markdown file with a fixed anatomy — copy one, change its values, reference it from SKILL.md, and Claude will pick it up on the next conversation.

stitclaude/houses/my-house.md 
# Duplicate the closest existing house
cp houses/swiss-quiet.md houses/my-house.md

# Edit the tokens, the voice, the canon references
$EDITOR houses/my-house.md

# Register it in SKILL.md — add one line
echo '- houses/my-house.md — when: [your context]' >> SKILL.md

Each house file must expose five sections: Palette, Typography, Motion, Do / Don't, Canon references. Keep the same schema as the shipped houses so Claude can reason about them uniformly.

#Contribute

Pull requests are welcome. Before submitting, run the 12-point checklist against your changes. If your PR adds a new house, include a showcase page in examples/ — the design must be demonstrable, not just described.

COC

Style, not opinion. Stit'Claude argues for rigor, not for any particular taste. A well-reasoned PR that widens the discipline is welcome. A PR that narrows it to personal preference is not.

#FAQ

Does Stit'Claude work with other LLMs?

The skill format is specific to Anthropic Claude (Haiku, Sonnet, Opus). The house descriptions and token tables are portable — any LLM can read them — but only Claude auto-loads them via the skills mechanism.

Why OKLCH for tokens?

Perceptual uniformity. Two tokens with the same L value read as equally bright, regardless of hue. See Field Experiment N° 002 for the demonstration with live sliders.

Can I use Stit'Claude without Claude?

Yes — read the houses as a manual, apply them by hand. You'll lose the automated discipline but gain the reference material. The components and tokens work as a standalone library.

Where is the license?

MIT. Fork it, ship it, do not misrepresent its authorship.
